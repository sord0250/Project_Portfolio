Cybersecurity compliance workflow

NCELT

NCELT is a local web app for organizing cybersecurity compliance evidence. It helps teams track files, map them to NIST CSF 2.0 outcomes, assign owners, and monitor review timing so evidence stays ready for audits, internal reviews, and GRC workflows.

Video placeholder

Problem

Why NCELT exists

Evidence collection often gets fragmented across shared drives, email threads, and spreadsheets. NCELT centralizes that work so a team can see what evidence exists, who owns it, how it maps to NIST CSF 2.0, and when it needs review again.

Solution

A practical GRC workflow

The app combines upload, ownership, framework mapping, and review timing into one local experience. That makes it easier to prepare for audits, keep stakeholders aligned, and avoid stale evidence slipping through review cycles.

Feature set

What users can do

  • Log in through manager-created accounts with session-based authentication
  • Upload evidence, assign an owner, and map one item to multiple NIST outcomes
  • Review the evidence dashboard for ownership, status, and next-review dates
  • Use the framework dashboard to spot missing or expiring evidence by outcome

AI-assisted development

How AI supported the build

I used AI to help shape the product direction and implementation path. I defined the relational data model and the core app criteria myself, including filtering, pages, and login behavior. I then used ChatGPT to think through the best tech stack, and Codex to implement the actual features, API routes, and app logic.

Product design

Data relationships, workflow criteria, filtering, page structure, and login requirements were designed before implementation.

Stack selection

ChatGPT was used to compare options and refine the stack for a local Python web app with authentication and dashboards.

Implementation

Codex helped build the concrete features, API routes, and app logic that turned the idea into a working product.

Implementation

Technical foundation

FastAPI handles routes, authentication, upload flows, and dashboards. SQLAlchemy and SQLite manage users, owners, outcomes, evidence records, and support relationships, while Jinja2 renders the interface and Passlib hashes passwords.

Lifecycle logic

How review status works

Each record stores a last-reviewed date and lifecycle length. NCELT calculates a next-review date from those fields and marks items green when current, yellow when approaching review, and red when past due or within three days of review.
